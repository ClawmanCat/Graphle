Common implementation for the BFS and DFS search algorithms, since their implementation is the same, just using a queue vs. a stack to keep track of the pending elements.

Template Parameters
STThe storage type used for the pending vertex list (DEQUE for BFS, VECTOR for DFS).
Parameters
graphA graphle::graph to visit.
rootThe root vertex to start the depth first search from.
visitorA visitor implementing the graphle::search_visitor interface.
emplaceAn object invocable as emplace(pending, element) to emplace the given element into the pending vertex list.
takeAn object invocable as take(pending) to take an element from the pending vertex list.
pending_providerAn optional storage-provider which can provide storage of type ST for the algorithm to use.
set_providerAn optional storage-provider which can provide a unordered-set-like type for the algorithm to use.
Returns
True if the algorithm finished normally or false if the visitor caused the algorithm to return early.
